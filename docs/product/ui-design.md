# FileShift UI 设计规范

## 1. 设计原则

- **简洁高效**：用户3秒内能找到目标功能
- **一致性**：所有页面遵循统一设计语言
- **响应式**：PC/平板/手机无缝适配
- **信任感**：专业色调+清晰的操作反馈

## 7. 核心组件规范

### 7.1 按钮(Button)

| 类型      | 样式             | 用途                    |
| --------- | ---------------- | ----------------------- |
| Primary   | 蓝色填充、白字   | 主要操作(开始转换/确认) |
| Secondary | 灰色边框、深色字 | 次要操作(取消/返回)     |
| Ghost     | 无边框、蓝字     | 辅助操作(查看更多)      |
| Danger    | 红色填充/边框    | 危险操作(删除)          |
| Disabled  | 灰色填充、浅灰字 | 不可操作状态            |

**尺寸**：

- Large: h-12, px-8, text-base（主要CTA）
- Medium: h-10, px-6, text-sm（常规按钮）
- Small: h-8, px-4, text-xs（紧凑场景）

### 7.2 上传区域(Upload Zone)

```
┌─────────────────────────────────────────┐
│                                         │
│          ☁️ (上传图标)                  │
│                                         │
│     拖拽文件到此处，或点击选择文件        │
│                                         │
│     支持 PDF, DOC, DOCX (最大20MB)      │
│                                         │
└─────────────────────────────────────────┘
```

- 虚线边框 `border-dashed`
- 拖拽时边框变蓝 + 背景高亮
- 圆角 `rounded-lg`
- 内边距 `p-12`

### 7.3 功能卡片(Tool Card)

```
┌──────────────────┐
│  🔄 (功能图标)   │
│                  │
│  PDF 转 Word     │  ← 功能名称 (font-semibold)
│                  │
│  将PDF转为可编辑  │  ← 简短描述 (text-gray-500)
│  Word文档        │
│                  │
│  消耗 3 积分     │  ← 积分提示 (text-sm)
└──────────────────┘
```

- 白色背景 + 阴影
- Hover时上浮效果 `hover:-translate-y-1`
- 圆角 `rounded-lg`
- 内边距 `p-6`

### 7.4 进度指示器(Progress)

- 线性进度条：上传进度、转换进度
- 环形进度：小型状态指示
- 骨架屏：页面加载中

## 9. 响应式断点

| 断点 | 宽度     | 设备          | 布局变化     |
| ---- | -------- | ------------- | ------------ |
| sm   | ≥ 640px  | 大屏手机/横屏 | 单列→双列    |
| md   | ≥ 768px  | 平板          | 侧边栏折叠   |
| lg   | ≥ 1024px | 小笔记本      | 完整导航显示 |
| xl   | ≥ 1280px | 桌面          | 最大内容宽度 |
| 2xl  | ≥ 1536px | 大屏          | 居中容器     |

### 移动端适配要点

- 导航栏折叠为汉堡菜单
- 功能卡片由多列变为单列/双列
- 上传区域全宽显示
- 底部固定操作按钮
- 触摸友好：按钮最小 44x44px

## 10. 动效规范

| 场景       | 动效       | 时长     |
| ---------- | ---------- | -------- |
| 按钮hover  | 背景色过渡 | 150ms    |
| 卡片hover  | 上浮+阴影  | 200ms    |
| 页面切换   | 淡入淡出   | 200ms    |
| 弹窗出现   | 缩放+淡入  | 250ms    |
| 进度条     | 平滑增长   | 300ms    |
| Toast通知  | 从顶部滑入 | 300ms    |
| 加载骨架屏 | 闪烁效果   | 1.5s循环 |

**原则**：动效服务于体验，不可过度使用。关键操作（上传/转换）需要明确的状态反馈。
